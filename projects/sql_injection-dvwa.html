DVWA SQL Injection – Login Bypass

Date: May 2025 | Category: Web App Pentesting

Platform: Damn Vulnerable Web Application (DVWA)

Vulnerability Type: SQL Injection (Authentication Bypass)

🧪 Objective

Demonstrate how to bypass DVWA login using SQL injection payloads on low security mode.

⚙️ Tools Used

  • Burp Suite
  • Firefox (localhost access)
  • MySQL (DVWA backend)

🖼️ Screenshots

Figure 1: Login page vulnerable to SQLi.

Figure 2: Intercepted request showing unvalidated input.

📌 Payloads Used

' OR 1=1 -- -
' OR '1'='1' #
admin' --

🧰 Exploitation Steps

  1. Launched DVWA locally and set security level to low.
  2. Accessed the vulnerable Login form.
  3. Submitted payload: ' OR 1=1 -- -.
  4. Authentication bypassed; access granted to admin dashboard.

📖 Explanation

The application constructs a vulnerable SQL query using unsanitized input:

SELECT * FROM users WHERE username = '' OR 1=1 -- - AND password = ''

Here, the OR 1=1 condition is always true, and the -- - sequence comments out the rest of the query, allowing the attacker to bypass authentication entirely.

🛡️ Mitigation Recommendations

  • Use parameterized queries or prepared statements.
  • Enforce strict server-side input validation and sanitization.
  • Deploy a Web Application Firewall (WAF) with SQLi rule sets.
